多态 ​

  1. 父类引用指向子类对象，子类实现父类提供的具体方法，也称向上转型
  2. 好处，解耦便于拓展，新增子类时只需实现具体方法即可
  3. 子类要继承或实现父类接口，重写里面的方法

Java的优势 ​

  1. 跨平台、兼容性强、生态丰富、工程化友好、自动内存管理
  2. 缺点：jvm内存大维护成本、编译启动较慢
  3. JVM是跨平台的关键，java代码编译成class字节码，jvm把字节码编译成各平台的机器码，java代码跨平台jvm不跨平台
  4. 三大特性：封装继承多态

Java中的参数传递是按值还是按引用 ​

  1. 不管是基本数据类型还是引用数据类型都是按照值
  2. 基本传值的副本、引用传引用的副本，因为传的是副本方法内修改都不会对原本的造成影响，引用的话指向的是另一个对象
  3. 栈内存存储基本数据类型、存储引用数据类型的在堆中的地址，堆中存的是引用数据类型的地址和整个对象
  4. 如果要修改外面变量的话，1.方法返回修改后的，原对象值来接收，2.传一个包含该值的对象，然后去改这个对象的那个值

不可变类 ​

  1. 条件：必须由final关键字修饰，防止子类去改变、变量必须声明为private final保证变量不可变、通过构造函数初始化所有字段、不提供任何setter和修改状态的方法
  2. 如String、Interger、Boolean等
  3. 优势: 天然支持多线程访问，不必担心被修改，适合作为缓存key因为不变性强
  4. 劣势：每次新建都得频繁的去创建对象，内存开销大，比如String拼接时都得去创建再去拼，所以后来有StringBuilder来替代这部分功能
  5. 为什么String要设计成不可变的，因为它大量用于类加载网络连接等敏感场景、二是因为字符串常量池存在的前提就是不可变、三是String作为HashMap的key比较频繁使用，hashCode用来缓存这样一次算完就可以了

Java为什么不支持多重继承 ​

  1. 继承菱形问题，B、C都继承了A（有方法hello）并重写了这个方法，D同时基础BC那么调用D的hello方法用的是B还是C的？这就有歧义，所以干脆就不让这样
  2. 为什么接口可以多实现，因为接口不定义具体实现，所以没啥可冲突的
  3. jdk8出现default接口也能去具体实现继承的方法，那么在多实现这类接口时也得指定调用哪个类的方法如B.super.hello()或C的

重载与重写的区别 ​

  1. 重载是对于一个类中，方法名一样，参数列表不同包括数量顺序等
  2. 重写是一个类继承另一个时，对其父类方法进行补充实现
  3. 重写时子类和父类能否抛出不同异常呢，可以但是要限制，不能子类实现中抛出的异常范围大于父类的

Java内部类的作用 ​

  1. 就是写在一个类中的另一个类包括成员内部类、静态内部类、局部内部类和匿名内部类
  2. 好处，内聚性，外部用不到的写在内部就好，对其他类的访问操作也很方便

Java8新特性 ​

  1. 新增函数式接口，只有一个抽象方法的接口
  2. 新的日期api，LocalDateTime等代替的Date这样线程不安全的类
  3. Lambda表达式，简化原本匿名内部类的冗长写法
  4. Stream api 提供数组相关处理
  5. Optional类，提供null判空的相关api等
  6. 新增了CompletableFuture这样的并发处理类

三个String类的区别 ​

  1. String不可变、StringBuffer/StringBuilder都是可变的，底层有一个可扩容的数组来存字符
  2. StringBuffer线程安全，由synchronized来维护适合多线程下频繁操作(synchronized加在方法上)，StringBuilder线程不安全适用于单线程
  3. 不一定多线程都要用StringBuffer，因为每个线程都有自己的StringBuilder示例，多个线程操作同一个可变字符串对象时草需要进行同步

StringBuilder是如何实现的 ​

  1. 本质是一个可扩容的字符数组 + count统计
  2. append添加时，先检查当前字符的字节大小，然后检查容量是否足够，不足则 *2+2扩容，然后拷贝内容到数组末尾，然后更新count
  3. 为啥是 * 2-均摊复杂度，为啥是 + 2处理边界情况-因为初始0的话 * 2就还是0没扩

包装类型和基本类型的区别 ​

  1. 基本类型有：int long short byte boolean double folat chat，包装类型则是对应的
  2. 性能上有差距，内存占用包装类型也大，包装类型==比较的是引用对象的地址值，基本就是值，所以对象的话equals比较
  3. 自动装箱和拆箱，就是包装类型和基本类型在jdk5之后都会自动转化
  4. 成员变量在堆，局部变量在栈

接口和抽象类有什么区别 ​

  1. 设计动机上，接口是自上而下的，我们知晓某一行为基于它来制定规则，实现该接口的类都要按照规则来，抽象类是自下而上的，从其他类中提取公共方法加以抽象复用
  2. 接口类中的方法和变量是public、abstaract，jdk8之后才可以通过duflut之类的来写里面的具体方法，抽象类中既可以定义abstaract抽象方法，也可以写具体方法
  3. 抽象类只能单继承，接口类可以多实现

Jdk和Jre的区别 ​

  1. jre是运行环境，包括jvm和最基本类库，不能编写代码
  2. jdk是包含jre，不仅可以运行代码也可以编写代码，且再次基础上新增了很多功能丰富的类库和开发工具
